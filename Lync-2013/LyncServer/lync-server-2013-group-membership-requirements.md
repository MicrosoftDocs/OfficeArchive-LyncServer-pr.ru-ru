# <a name="group-membership-requirements-for-lync-server-2013"></a>Требования членства в группах для Lync Server 2013

_**Тема последнего изменения:** 2012-10-05_

В приведенной ниже таблице перечислены группы или группы, которым должен принадлежать человек, чтобы успешно установить, управлять и устранять неполадки в Lync Server 2013.

<colgroup>
<col style="width: 50%" />
<col style="width: 50%" />

<thead>
<tr class="header">
<th>Исполняемый файл Lync Server 2013</th>
<th>Требуется членство в группах</th>
</tr>

<tr class="odd">
<td><p><strong>Setup.exe</strong> — исполняемый файл, который запускает установку средств администрирования Lync Server 2013.</p></td>
<td><p>Входить в локальную группу администраторов на компьютере, с которого запускается исполняемый объект. Член группы "Пользователи домена" для чтения данных в доменных службах Active Directory. Этот уровень разрешений необходим, поскольку автоматическая установка требуемых пакетов MSI на локальном компьютере требует наличия привилегий для чтения и записи на защищенные локальные ресурсы компьютера, такие как каталоги программных файлов, и защищенный реестр, например куст локального компьютера.</p>
<div>

> [!TIP]  
> Вы также можете делегировать разрешения на настройку для пользователей или групп, которым вы не хотите предоставлять членство в группе Администраторы домена. Подробности можно найти <A href="lync-server-2013-granting-setup-permissions.md">в разделе Предоставление разрешений на установку в Lync Server 2013</A> в документации по развертыванию.


</div></td>
</tr>
<tr class="even">
<td><p><strong>Deploy.exe</strong> вызывается setup.exe, deploy.exe отвечает за развертывание компонентов программного обеспечения для ролей сервера.</p></td>
<td><p>Входить в локальную группу администраторов на компьютере, с которого запускается исполняемый объект. Член группы "Пользователи домена" для чтения данных в доменных СЛУЖБах Active Directory. Этот уровень разрешений необходим, поскольку автоматическая установка требуемых пакетов MSI на локальном компьютере требует наличия привилегий для чтения и записи на защищенные локальные ресурсы компьютера, такие как каталоги программных файлов, и защищенный реестр, например куст локального компьютера. Членство в группе RtcUniversalReadOnlyAdmins необходимо для чтения хранилища центрального управления.</p>
<div>

> [!NOTE]  
> Если вы используете операционную систему Windows Vista или операционную систему Windows 7, вам будет предложено управлять учетными записями пользователей (UAC) для продолжения установки. Если вы вошли в систему с помощью стандартной учетной записи пользователя, вам понадобится кто-то, кто входит в локальную группу администраторов для предоставления учетных данных, если вам будет предложено ввести учетную запись с разрешениями на установку программного обеспечения.


</div></td>
</tr>

<td><p><strong>Bootstrapper.exe</strong> — вызывается setup.exe, bootstrapper.exe отвечает за развертывание и настройку ролей сервера.</p></td>
<td><p>Входить в локальную группу администраторов на компьютере, с которого запускается исполняемый объект. Участник группы RTCUniversalServerAdmins для выполнения Bootstrapper.exe. Член группы "Пользователи домена" для чтения данных в доменных СЛУЖБах Active Directory. Этот уровень разрешений необходим, поскольку автоматическая установка требуемых пакетов MSI на локальном компьютере требует наличия привилегий для чтения и записи на защищенные локальные ресурсы компьютера, такие как каталоги программных файлов, и защищенный реестр, например куст локального компьютера.</p></td>
</tr>
<tr class="even">
<td><p><strong>TopologyBuilder</strong> — пользовательский интерфейс, управляемый мастером, для создания, просмотра, настройки и проверки топологий Lync Server 2013.</p></td>
<td><p>Член локальной группы администраторов на компьютере, с которого запускается исполняемый объект для просмотра топологии. Чтобы изменить параметры конфигурации, пользователь должен быть членом группы RTCUniversalServerAdmins. Чтобы опубликовать топологию, участник группы RTCUniversalServerAdmins и группа администраторов домена или член группы RTCUniversalServerAdmins (только если группе предоставлены разрешения на настройку делегирования). Дополнительные сведения о делегировании разрешений на настройку для разрешения пользователям, которым разрешено публиковать топологию, без участия в группе Администраторы домена, приведены в разделе <a href="lync-server-2013-granting-setup-permissions.md">предоставление разрешений на установку в Lync Server 2013</a> в документации по развертыванию.</p></td>
</tr>
<tr class="odd">
<td><p><strong>AdminUIHost</strong> — графический пользовательский интерфейс на основе веб-сайта для управления Lync Server 2013.</p></td>
<td><p>Участник группы CsAdministrator или участник другой роли управления доступом на основе ролей (RBAC), которым назначена определенная административная задача. Панель управления Lync Server 2013 реализует изменения конфигурации с помощью командлетов командной консоли Lync Server 2013. Список стандартных ролей и членов командлетов, которые разрешено запускать, можно найти <a href="lync-server-2013-planning-for-role-based-access-control.md">в разделе Планирование управления доступом на основе ролей в Lync Server 2013</a> в документации по планированию.</p></td>
</tr>
<tr class="even">
<td><p><strong>PowerShell.exe с помощью модуля Lync Server 2013, загруженного</strong> с помощью средства администрирования для командной строки, с командлетами, предназначенными для управления Lync Server 2013.</p></td>
<td><p>Участник группы CsAdministrator или член другой роли RBAC, которому назначен определенный командлет. Список стандартных ролей и членов командлетов, которые разрешено запускать, можно найти <a href="lync-server-2013-planning-for-role-based-access-control.md">в разделе Планирование управления доступом на основе ролей в Lync Server 2013</a> в документации по планированию.</p>

<li><p>RTCUniversalUserAdmins</p></li>
<li><p>RTCUniversalReadOnlyAdmins</p></li>
</ul></td>
